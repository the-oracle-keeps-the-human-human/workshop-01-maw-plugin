## Lessons Learned (บทเรียนและสิ่งที่เราเรียนรู้)

### 1. การทำงานร่วมกันภายใต้กฎความเงียบและการส่งต่อสาร (Silence & Delegation Rules)
การที่ทีมบอทมีการแบ่งแยกขอบเขตงานอย่างชัดเจน (Target-Specific Delegation) ทำให้สารต่างๆ จากมนุษย์ที่จ่าหน้าเฉพาะเจาะจงไม่เกิดสภาวะแย่งกันตอบ (No-collision) และการที่ No.8 นิ่งเงียบบนหน้า Discord แต่ทำหน้าที่ส่งต่อสารหลังบ้านผ่าน `maw hey 06-gemini` ทำให้สามารถประสานการทำงานและรักษาความน่าเชื่อถือของสภาออราเคิลได้อย่างมั่นคง

### 2. การกู้คืนระบบจาก Process Deadlock & Watchdog (Resilience and Operations)
การประมวลผลข้อมูลขนาดใหญ่ เช่น หน้าเว็บ HTML ของ issue อาจนำไปสู่สภาวะค้าง (Deadlock) ได้ง่าย ในกรณีนี้การมี No.10 X คอยเฝ้าสังเกตการณ์ สแกน process และใช้วิธีปลดบล็อกด้วยการฆ่า process (PID 1785015) เพื่อเปิดทางให้ระบบ watchdog รีสตาร์ทการทำงานอย่างสะอาด เป็นสิ่งที่ช่วยเสริมสร้างความแข็งแกร่งและความทนทานให้กับระบบหลังบ้าน (Operations Resilience)

### 3. การออกแบบ CLI Plugin ในรูปแบบ Dual-Mode
ในการพัฒนาปลั๊กอิน `agy-nano2` เราได้ออกแบบให้ `index.ts` รองรับการรันแบบสองโหมด (Dual-Mode) ได้แก่:
- **Workshop SDK Mode**: สำหรับให้ตัวรันเวิร์กชอปของโรงเรียนเรียกใช้ผ่าน `api.command`
- **Local CLI Context Mode**: สำหรับการเรียกใช้ผ่าน SDK ยุคใหม่ `InvokeContext`
ตรงนี้ช่วยให้ปลั๊กอินสามารถทำงานได้อย่างสมบูรณ์แบบทั้งการส่งงานในระบบของโรงเรียนและการรันสอบทานเองภายในเครื่อง

### 4. การจัดการ Shared State บนระบบ Multi-User/Agent
การใช้ไฟล์สถานะร่วมกันอย่าง [focus.md](file:///root/ψ/focus.md) แม้จะเกิดโอกาสทับซ้อนกันบ้าง (Race condition) ในช่วงเวลาที่บอททำงานพร้อมกัน แต่การที่บอททุกตัวขยันเข้ามาบันทึกสถานะการทำกิจกรรมและคอยสแกนดูประวัติการทำงานใน [activity.log](file:///root/ψ/activity.log) เสมอ ทำให้ทีมสามารถปรับทิศทางและต่อประสานงานกันได้อย่างรวดเร็ว
